# Handoff Contracts

Use this reference when reading is a stage before another DBX skill.

## 1. Handoff to `dbx-learn`

Use when the next job is durable understanding, practice reps, quiz/review, or learning records.

```markdown
## Handoff to dbx-learn
- Learning target:
- Source(s):
- Coverage:
- Core model from reading:
- Misconceptions or traps:
- Practice candidates:
- Active recall candidates:
- Open questions:
- Caveats:
```

Do not claim the user learned it just because a reading note exists.

## 2. Handoff to `dbx-write`

Use when the next job is a viewpoint-driven article, technical blog, or public Markdown draft.

```markdown
## Handoff to dbx-write
- Writing seed:
- Source facts available:
- Claims safe to use:
- Claims needing verification:
- Strong examples:
- Tensions / trade-offs:
- Must not overclaim:
- Suggested article angle:
```

Keep source facts separate from the future article thesis.

## 3. Handoff to decision / judgment skills

Use when the source becomes evidence for a decision, plan, review, product/design judgment, or architecture critique.

```markdown
## Judgment handoff
- Decision or judgment question:
- Source evidence:
- Assumptions:
- Risks:
- Unknowns:
- Suggested DBX skill:
- Minimum extra evidence needed:
```

## 4. Handoff to `dbx-attention-routing`

Use only when reading one source reveals a broader queue problem, or when the user supplies many mixed items.

```markdown
## Attention routing handoff
- Items:
- Visible evidence per item:
- Candidate routes, if obvious:
- Profile assumptions:
- Unsafe or high-risk items:
```

## 5. Handoff failure

If the reading source was not sufficiently extracted, do not hand off confident claims. Use:

```markdown
## Incomplete handoff
- Source attempted:
- Failure state:
- What is safe to say:
- What must be re-fetched or provided by user:
```
