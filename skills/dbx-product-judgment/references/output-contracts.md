# Product Judgment Output Contracts

Use the default contract in `SKILL.md` unless one of these smaller or larger shapes fits better.

## Compact quick read

```markdown
## 判断
- 结论：...
- 置信度：...
- 一句话原因：...

## 最关键的问题
1. [P1 high] ...
   - Evidence: ...
   - Impact: ...
   - Fix: ...

## 下一步
- 先验证/修改：...
- 我没有验证：...
```

## Standard product audit

```markdown
## 核心判断
- 结论：做对 / 方向对但未证明 / 局部成立但关键风险未解 / 不建议继续 / 信息不足无法判断
- 置信度：high / medium / low
- 判断对象：...
- 最主要风险：...

## 范围和证据
- 已查看：...
- 未查看：...
- 使用的证据：...
- 假设：...
- 未验证：...

## 产品模型
- 目标用户：...
- 核心场景：...
- 用户要完成的状态变化：...
- 当前关键路径：...
- 成功标准：...

## 主要发现
1. [P1 high] 标题
   - Evidence: ...
   - Impact: ...
   - Fix: ...
   - Validation: ...
   - Confidence: ...

## 优先级建议
- 先做：...
- 暂缓：...
- 不做：...

## 验证计划
- 定性验证：...
- 定量验证：...
- 技术验证：...
- Guardrail：...

## 仍需确认的问题
- ...
```

## Insufficient context

```markdown
## 暂不能下结论
当前缺少会改变判断的背景：...

## 我能基于现有信息判断的部分
- ...

## 需要补充的最少问题
1. ...
2. ...
3. ...
```

Use this when target user, product object, evidence boundary, or judgment target is missing.

## Saved report validation

When the report is saved to Markdown, optionally run:

```bash
python3 skills/dbx-product-judgment/scripts/validate-product-report.py path/to/report.md
```

This checker only verifies handoff shape and evidence markers. It does not prove the product judgment is correct.
