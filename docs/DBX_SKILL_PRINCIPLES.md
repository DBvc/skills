## DBX Working Principles

1. Reduce repeated task entropy.
2. Define boundaries before capability.
3. Keep claims evidence-bounded.
4. Place controls in the right artifact.
5. Evolve through feedback and regression.
6. Treat skill collections as systems.
7. Make persistent state explicit when present.

Do not paste this file into runtime skills. Convert only the relevant principle into operational behavior.
